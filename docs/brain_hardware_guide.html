Buying a box to host your Brain

What to look for on the market if you want to run your own personal AI “brain” at home instead of on the cloud. Figures below are measured on the current system, not estimates.

Updated 2026-06-14 For: self-hosting the memory brain

Your “brain” is a small, always-on appliance that stores your memories and answers semantic searches over them. Four things run on it:

ComponentJobAppetite
PostgreSQL + pgvectorStores memories and their 1024-dim vectors Light. ~4 KB per memory; 50,000 memories ≈ 200 MB of vectors
Embedding model (bge-m3 via Ollama)Turns text into vectors for search The main RAM user: ~1–2 GB resident. CPU is fine — no GPU needed
Memory service (Python)The API the platform talks to ~200–400 MB
Tunnel agent (e.g. Tailscale/WireGuard)Lets the platform reach the box behind your router Negligible. No port-forwarding needed — the box dials out

The one rule that decides everything: RAM is king, and a GPU only matters if you want the box to also generate text locally (run an LLM). For memory + search, embeddings run fine on a plain CPU. Don’t pay for a graphics card you won’t use.

What we measured (so the numbers are honest)

Embedding model footprint~1.2 GB resident in RAM (bge-m3)
First request after idle (“cold”)~12 s — loads the model into RAM
Subsequent requests (“warm”)~0.8 s each on 2 modest CPU cores
Test box: 2 vCPU / 4 GB RAMToo small — ran out of memory when the model loaded alongside the apps. Needed swap to survive.

Takeaway: 4 GB is the floor that breaks. Plan for headroom — see tiers below.

Pick a tier

TIER 1 · START HERE

Essential Brain

Memory + search fully local. Text generation (Claude-style answers) still uses a cloud API.

  • RAM: 8 GB min · 16 GB recommended
  • CPU: any modern 4-core (x86-64 or Apple Silicon)
  • GPU: none
  • Storage: 256 GB NVMe SSD (512 GB comfy)
  • Power: ~7–15 W idle
TIER 2 · PRIVATE GENERATION

Sovereign Brain

Everything local, including a small (7–13B) language model. Nothing leaves the box.

  • RAM: 16–32 GB (or 16–24 GB Apple unified)
  • CPU: 6+ cores
  • GPU: NVIDIA 8–12 GB VRAM, or Apple Silicon unified memory
  • Storage: 512 GB–1 TB NVMe
  • Power: ~20–60 W
TIER 3 · POWER

Power Brain

Bigger/faster local models, snappy responses, room to grow into more features.

  • RAM: 32–64 GB
  • CPU: 8+ cores
  • GPU: NVIDIA 16–24 GB VRAM (e.g. 3090/4090), or Apple 48–64 GB
  • Storage: 1–2 TB NVMe
  • Power: 60–150 W+

Most people want Tier 1 to start. Step up only if “the AI must generate on my own hardware too” matters to you — that’s the expensive, GPU/RAM-hungry part. The brain’s core job (remember + search) is cheap.

Which form factor?

Mini PC (x86)

Beelink, Minisforum, GMKtec, Intel NUC. Cheap, tiny, low-power, RAM/SSD usually upgradeable. Best value for Tier 1. Limited for local LLMs (no real GPU).

Mac mini (Apple Silicon)

Silent, very efficient, and its unified memory doubles as “VRAM” — excellent for Tier 1 and Tier 2 local LLMs. RAM is not upgradeable, so buy 16–24 GB+ up front.

Docker-capable NAS

Synology / UGREEN / TrueNAS. Convenient if you already own one and it’s always on. Check it can run containers and has ≥ 16 GB RAM.

Small tower + NVIDIA GPU

The flexible, upgradeable route for Tier 2/3 local generation. Louder, more power. VRAM is the spec that matters — see below.

What to actually check on the listing

  • RAM ≥ 16 GB. This is the number that bit us at 4 GB. Upgradeable is a bonus on x86; fixed on Apple — so buy up front there.
  • NVMe SSD ≥ 512 GB. Databases want fast random I/O. Avoid HDD and eMMC as the main drive.
  • If (and only if) you want local generation: the gating spec is NVIDIA VRAM (8 GB runs 7–8B, 12–16 GB runs ~13B, 24 GB runs much larger/faster) or Apple unified-memory size. For embeddings only, ignore the GPU entirely.
  • Wired Gigabit Ethernet and the ability to stay always-on.
  • Low idle power & quiet — it lives in your home and runs 24/7.
  • CPU: x86-64 or Apple Silicon both work. 64-bit, supports containers/virtualization.

Avoid

  • Raspberry Pi / low-end ARM SBCs — too little RAM/CPU for the embedding model to be pleasant, and a non-starter for local LLMs.
  • Anything with < 8 GB RAM (and treat 8 GB as the bare minimum, not the target).
  • eMMC-only or spinning-disk-only storage for the database.
  • A pricey GPU “to be safe” if you only want memory + search — you won’t use it.

Operating notes

Networking

The box only needs outbound internet. It dials out and joins a private mesh (e.g. Tailscale), so the platform can reach it without you opening any ports on your router. Wired connection preferred.

Reliability

It’s an appliance: headless, always-on. A small UPS is nice so a power blip doesn’t corrupt the database. Keep a backup of the Postgres data — that is your brain.

TL;DR shopping line: a quiet mini-PC or Mac mini with 16 GB RAM and a 512 GB NVMe SSD, wired ethernet, low idle power — no GPU. Add a GPU (or buy Apple with more unified memory) only when you decide the AI’s writing, not just its memory, must run on your own hardware too.
